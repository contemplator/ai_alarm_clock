# AI 新聞鬧鐘 App - 產品需求文件 (PRD)

> **版本**: 1.0.0  
> **最後更新**: 2025-12-07  
> **狀態**: 草稿

---

## 1. 產品概述

### 1.1 產品願景

一款結合智慧新聞播報的鬧鐘 App，讓使用者在起床時能夠透過語音接收當日新聞摘要，以更有效率的方式開始一天。

### 1.2 目標使用者

- 習慣早起接收資訊的上班族
- 想在起床時快速了解時事的使用者
- 希望用語音取代傳統鬧鈴的使用者

### 1.3 核心價值主張

- **省時**：起床同時接收新聞，不需額外花時間閱讀
- **智慧**：AI 自動摘要，只聽重點
- **漸進喚醒**：先播鬧鈴，再播語音，溫和喚醒

## 2. 功能需求

### 2.1 核心功能 (MVP)

#### 2.1.1 鬧鐘管理

| 功能     | 描述                                            | 優先級 |
| -------- | ----------------------------------------------- | ------ |
| 新增鬧鐘 | 使用者可設定多組鬧鐘                            | P0     |
| 時間設定 | 選擇時、分                                      | P0     |
| 重複設定 | 支援：每日、工作日(週一~五)、週末(週六日)、自訂 | P0     |
| 開關鬧鐘 | 可獨立開啟/關閉每組鬧鐘                         | P0     |
| 刪除鬧鐘 | 可刪除鬧鐘                                      | P0     |
| 編輯鬧鐘 | 可修改已建立的鬧鐘設定                          | P0     |

#### 2.1.2 鬧鈴設定

| 功能         | 描述                                     | 優先級 |
| ------------ | ---------------------------------------- | ------ |
| 系統鈴聲選擇 | 從 App 內建鈴聲中選擇                    | P0     |
| 鈴聲預覽     | 選擇時可試聽                             | P1     |
| 鈴聲時長     | 固定播放約 15 秒後自動停止並進入新聞播報 | P0     |
| 音量設定     | 可調整鬧鈴音量                           | P1     |

#### 2.1.3 新聞播報

| 功能     | 描述                                    | 優先級 |
| -------- | --------------------------------------- | ------ |
| 新聞來源 | Google 新聞、MSN 新聞、Yahoo 新聞 (RSS) | P0     |
| AI 摘要  | 使用 AI 將新聞內容摘要成語音適合的長度  | P0     |
| 語音播報 | 使用系統 TTS 播放摘要                   | P0     |
| 新聞數量 | 可設定播報 3~10 則新聞                  | P1     |
| 新聞類別 | 可選擇新聞類別（財經、科技、國際等）    | P2     |

#### 2.1.4 鬧鐘觸發流程

```
時間到達 → 播放鬧鈴(15秒) → 自動切換至新聞播報 → 播報完畢 → 鬧鐘結束
                ↓
           使用者可隨時點擊「停止」結束鬧鐘
```

### 2.2 進階功能 (V2+)

| 功能         | 描述                        | 優先級 |
| ------------ | --------------------------- | ------ |
| 天氣播報     | 新聞前先播報今日天氣        | P2     |
| 個人化新聞   | 根據閱讀習慣推薦            | P3     |
| 雲端同步     | 多裝置同步鬧鐘設定          | P3     |
| 語音品質升級 | Google Cloud TTS 高品質語音 | P3     |
| 會員系統     | 進階功能需登入              | P3     |

## 3. 非功能需求

### 3.1 效能需求

- 鬧鐘觸發延遲 < 1 秒
- 新聞摘要生成 < 5 秒（可預先快取）
- App 啟動時間 < 2 秒
- 背景運行時電池消耗最小化

### 3.2 可靠性需求

- 鬧鐘必須 100% 可靠觸發（即使 App 在背景或裝置重啟後）
- 新聞獲取失敗時應有 fallback（播放預設音樂或通知使用者）
- 支援離線模式（至少鬧鈴功能可正常運作）

### 3.3 安全性需求

- API Key 安全儲存（Keychain）
- 使用者資料本地儲存，不上傳至第三方（MVP 階段）

### 3.4 相容性需求

- 支援 iOS 15.0+
- 支援所有 iPhone 機型（iPhone SE 以上）

## 4. 使用者故事

### 4.1 設定鬧鐘

```
作為一位使用者，
我希望能夠設定多個鬧鐘，
並為每個鬧鐘選擇不同的重複週期，
以便我可以在工作日和週末有不同的起床時間。
```

### 4.2 體驗新聞播報

```
作為一位使用者，
當鬧鐘響起時，
我希望先聽到 15 秒的鬧鈴，
然後自動開始播報今日新聞摘要，
以便我在起床時能獲取最新資訊。
```

### 4.3 自訂新聞來源

```
作為一位使用者，
我希望能夠選擇我偏好的新聞來源，
以便聽到我感興趣的新聞內容。
```

### 4.4 停止鬧鐘

```
作為一位使用者，
當我想停止鬧鐘或新聞播報時，
我可以點擊停止按鈕，
讓鬧鐘立即結束。
```

## 5. 限制與假設

### 5.1 限制

- iOS 背景執行有嚴格限制，需使用 Local Notification + Background Modes
- AI 摘要需要網路連線
- 系統 TTS 品質有限，但為免費方案

### 5.2 假設

- 使用者起床時有網路連線
- 使用者接受系統 TTS 的語音品質
- RSS Feed 為穩定可用的新聞來源
